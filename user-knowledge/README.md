# user-knowledge/

The single source of truth about the candidate. The assistant reads these files
when it generates, tailors, or evaluates a resume, and writes to them whenever the
user shares new durable facts (see `.claude/rules/user-knowledge.md`).

Fill in the files below. Keep entries terse, factual, and ASCII only (no em dashes).
Only record what is genuinely true; mark anything in progress or uncertain.

| File | Holds |
|------|-------|
| `profile.md` | Name, contact, links, headline, summary themes |
| `experience.md` | Jobs and clients with dates, scope, accomplishments, metrics |
| `projects.md` | Projects with stack and standout details |
| `skills.md` | Skills actually used, grouped by category |
| `education.md` | Degrees and schools |
| `certifications.md` | Certifications (note any in progress) |
| `preferences.md` | Formatting and tone preferences |
